## 💡연산자의 종류

  ````
산술 연산자 : 숫자 데이터를 대상으로 하는 연산을 수행합니다. 주로 덧셈(+), 뺄셈(-), 곱셈(*), 나눗셈(/), 나머지(%) 등이 있습니다.
비교 연산자 : 두 값을 비교하여 관계를 평가하고 논리적인 결과를 반환합니다. 예를 들어, ==, ===, !=, !==, >, &lt;, >=, &lt;= 등이 있습니다.
단항 연산자 : 가장 일반적인 단항 연산자는 부호를 바꾸는 "-(음수)" 연산자와 값을 반대로 하는 "!"(논리 부정) 연산자입니다.
논리 연산자 : 주로 조건문과 논리적인 표현식을 평가하여 참(true) 또는 거짓(false)을 결정하는 데에 활용됩니다.
삼항 연산자 : 조건에 따라 값을 선택하여 반환합니다. 주로 condition ? expr1 : expr2 형태로 사용되며, 조건이 참이면 expr1을, 거짓이면 expr2를 반환합니다.
대입 연산자 : 대입 연산자는 대개 "=" 기호로 표시되며, 왼쪽에는 변수가 오고 오른쪽에는 값을 표현하는 표현식이나 값이 옵니다. 이를 통해 변수에 값을 할당하게 됩니다.
비트 연산자 : 비트 연산자는 주로 비트 단위의 논리 연산, 시프트(이동), 비트 단위의 AND, OR, XOR 연산 등을 수행하는 데에 활용됩니다.
````
    
## 💡연산자의 우선순위

  ````
1. 산술 > 비교 > 논리 > 대입, 대입은 제일 마지막에 수행된다.
2. 단항(1) > 이항(2) > 삼항(3). 단항 연산자의 우선순위가 이항 연산자보다 높다.
3. 단항 연산자와 대입 연산자를 제외한 모든 연산의 진행방향은 왼쪽에서 오른쪽이다.
````

## 💡산술 연산자: 프로그래밍에서의 기초적인 수학 작업

  ````
프로그래밍에서 산술 연산자는 가장 기본적이면서도 중요한 요소 중 하나입니다. 이 연산자들은 수학적인 계산을 수행하고, 프로그램에서 숫자를 조작하고 조건을 처리하는 데에 사용됩니다.

덧셈(+): 덧셈 연산자는 두 값을 더합니다. 예를 들어, 2 + 3은 5가 됩니다. 이 연산자는 값을 결합하고 두 값의 합을 반환합니다.

뺄셈(-): 뺄셈 연산자는 첫 번째 값을 두 번째 값에서 빼는 연산을 수행합니다. 예를 들어, 5 - 3은 2가 됩니다. 이 연산자는 값의 차이를 반환합니다.

곱셈(*): 곱셈 연산자는 두 값을 곱합니다. 예를 들어, 2 * 3은 6이 됩니다. 이 연산자는 값을 반복하여 더하거나 그룹화된 값의 크기를 변경하는 데 사용됩니다.

나눗셈(/): 나눗셈 연산자는 첫 번째 값을 두 번째 값으로 나눕니다. 예를 들어, 6 / 2는 3이 됩니다. 이 연산자는 값의 비율을 나누거나 분할하는 데 사용됩니다.

나머지(%): 나머지 연산자는 첫 번째 값을 두 번째 값으로 나눈 후 나머지를 반환합니다. 예를 들어, 7 % 3은 1이 됩니다. 이 연산자는 숫자를 그룹화하거나 순환적인 작업을 수행할 때 유용합니다.

거듭제곱( 또는 ^)**: 거듭제곱 연산자는 첫 번째 값을 두 번째 값의 지수로 사용하여 거듭제곱을 계산합니다. 예를 들어, 2 ** 3은 8이 됩니다. 이 연산자는 값의 거듭제곱을 계산하는 데 사용됩니다.

산술 연산자는 프로그래밍 언어의 기초이며, 숫자형 데이터를 처리하는 데에 필수적입니다. 따라서 이러한 연산자를 잘 이해하고 활용하는 것은 프로그래밍을 시작하는 데 중요한 요소입니다.
````

## 💡비교 연산자: 값의 비교와 조건 처리

  ````
프로그래밍에서 비교 연산자는 두 값을 비교하여 논리적인 참(true) 또는 거짓(false)을 반환하는 데 사용됩니다. 이를 통해 프로그램은 조건을 판단하고 특정한 작업을 수행할 수 있습니다. 주로 다음과 같은 비교 연산자들이 사용됩니다.

동등 비교(==): 두 값이 서로 같은지 비교합니다. 예를 들어, 5 == 5는 참(true)이 됩니다. 이 연산자는 값의 동등 여부를 확인합니다.

부등 비교(!=): 두 값이 서로 다른지 비교합니다. 예를 들어, 5 != 3은 참(true)이 됩니다. 이 연산자는 값의 부등 여부를 확인합니다.

크기 비교(>, <, >=, <=): 두 값을 크기로 비교합니다. ">" 연산자는 왼쪽 값이 오른쪽 값보다 큰지를 확인하며, "<" 연산자는 왼쪽 값이 오른쪽 값보다 작은지를 확인합니다. ">=" 연산자는 왼쪽 값이 오른쪽 값보다 크거나 같은지를 확인하며, "<=" 연산자는 왼쪽 값이 오른쪽 값보다 작거나 같은지를 확인합니다. 예를 들어, 5 > 3은 참(true)이 됩니다. 이러한 연산자들은 값의 크기를 비교하여 결과를 반환합니다.

비교 연산자는 조건문(if, while 등)에서 주로 사용되며, 프로그램의 흐름을 제어하는 데에 중요한 역할을 합니다. 값의 비교를 통해 프로그램은 다양한 조건에 따라 다른 작업을 수행할 수 있습니다. 이러한 연산자들을 잘 이해하고 활용하는 것은 프로그래밍에서 필수적인 기술입니다.
````

## 💡단항 연산자: 변수와 값의 조작

````
프로그래밍에서 단항 연산자는 하나의 피연산자에 대해 작동하는 연산자로, 변수나 값을 조작하는 데에 사용됩니다. 이 연산자들은 주로 수학적인 연산 또는 값을 변경하는 데에 활용됩니다. 다음은 주요한 단항 연산자들에 대한 설명입니다.

부호 변경 (-): 단항 "-" 연산자는 숫자의 부호를 반대로 바꿉니다. 예를 들어, "-5"는 5의 반대인 -5를 나타냅니다. 이 연산자는 변수나 값의 부호를 변경할 때 사용됩니다.

논리 부정 (!): 단항 "!" 연산자는 논리값을 반전시킵니다. 예를 들어, 만약 변수 x가 true라면, !x는 false가 되고, 반대로 만약 변수 x가 false라면, !x는 true가 됩니다. 이 연산자는 조건문에서 부정적인 조건을 만들거나 논리값을 반대로 변경할 때 사용됩니다.

증가 및 감소 (++/--): 단항 "++"와 "--" 연산자는 변수의 값을 1씩 증가시키거나 감소시킵니다. 예를 들어, "x++"는 변수 x의 값을 1 증가시키고, "--y"는 변수 y의 값을 1 감소시킵니다. 이 연산자들은 반복문에서 변수를 증가시키거나 감소시키는 데에 사용됩니다.

비트 단순화 (~): 단항 "~" 연산자는 비트를 반전시킵니다. 즉, 0은 1로, 1은 0으로 바꿉니다. 이 연산자는 비트 단위 연산을 수행할 때 사용됩니다.

데이터 형 변환: 몇몇 언어에서는 단항 연산자를 사용하여 데이터 형을 변환할 수 있습니다. 예를 들어, "int(x)"는 변수 x를 정수형으로 변환합니다. 이를 통해 데이터의 형태를 필요에 맞게 변환할 수 있습니다.

이러한 단항 연산자들은 프로그래밍에서 변수와 값의 조작을 위해 필수적입니다. 올바르게 이해하고 활용한다면, 변수와 값을 더욱 유연하게 다룰 수 있으며, 코드의 가독성과 효율성을 향상시킬 수 있습니다.
````

## 💡논리 연산자: 프로그래밍에서의 조건 판단과 흐름 제어

  ````
프로그래밍에서 논리 연산자는 조건을 판단하고 프로그램의 흐름을 제어하는 데에 중요한 도구입니다. 논리 연산자는 주로 논리적인 조건을 평가하고 그 결과에 따라 코드 블록을 실행하거나 건너뛰는 데에 사용됩니다. 다음은 주요한 논리 연산자들에 대한 설명입니다.

논리 AND (&&): 논리 AND 연산자는 두 개의 조건이 모두 참일 때만 전체 표현식이 참이 됩니다. 즉, 조건 A와 조건 B가 모두 참일 때 전체 표현식은 참이 됩니다. 이 연산자는 모든 조건이 충족되어야 하는 경우에 사용됩니다.

논리 OR (||): 논리 OR 연산자는 두 개의 조건 중 하나 이상이 참일 때 전체 표현식이 참이 됩니다. 즉, 조건 A 또는 조건 B 중 하나 이상이 참일 때 전체 표현식은 참이 됩니다. 이 연산자는 하나 이상의 조건을 충족해야 하는 경우에 사용됩니다.

논리 NOT (!): 논리 NOT 연산자는 피연산자의 논리적인 상태를 반전시킵니다. 즉, 참이면 거짓으로, 거짓이면 참으로 변경됩니다. 이 연산자는 조건을 부정하는 데에 사용됩니다.

논리 연산자는 주로 조건문(if, while 등)에서 사용되며, 프로그램의 흐름을 제어하는 데에 중요한 역할을 합니다. 이를 통해 프로그램은 다양한 상황에 대처하고 유연하게 동작할 수 있습니다. 따라서 논리 연산자를 잘 이해하고 활용하는 것은 프로그래밍에서 필수적입니다.
````

## 💡삼항 연산자: 간결하고 유용한 조건 표현

  ````
프로그래밍에서 삼항 연산자는 조건문을 간결하게 표현하고 코드의 가독성을 향상시키는 데에 유용한 도구입니다. 삼항 연산자는 조건에 따라 두 가지 결과 중 하나를 선택하여 반환하는 조건부 표현식입니다. 다음은 삼항 연산자의 구조와 활용에 대한 설명입니다.

삼항 연산자는 다음과 같은 구조를 가집니다.

조건이 참일 때에는 참일 때 값이 선택되고, 조건이 거짓일 때에는 거짓일 때 값이 선택됩니다.
````

## 💡대입 연산자: 변수에 값을 할당하는 프로그래밍의 핵심

  ````
프로그래밍에서 대입 연산자는 변수에 값을 할당하는 데 사용되는 중요한 요소입니다. 이 연산자는 변수에 값을 저장하고 변수의 값을 변경하는 데에 사용됩니다. 대부분의 프로그래밍 언어에서는 "=" 기호를 사용하여 대입 연산을 수행합니다. 다음은 대입 연산자의 기능과 활용에 대한 설명입니다.

기능:
대입 연산자는 변수에 값을 할당하는 기능을 수행합니다. 이는 프로그램에서 변수를 사용하여 데이터를 저장하고 조작하는 데에 필수적입니다. 대입 연산자를 사용하여 변수에 값을 할당함으로써 변수를 초기화하거나 값을 변경할 수 있습니다.

활용:

1.변수 초기화:
대입 연산자는 변수를 초기화하는 데에 사용됩니다. 변수를 선언한 후 처음 값을 할당할 때 사용됩니다.

2.값의 변경:
변수에 새로운 값을 할당하여 값을 변경하는 데에도 대입 연산자가 사용됩니다.

3.복합 대입 연산자:
대입 연산자는 다른 연산과 결합하여 사용될 수 있습니다. 이러한 연산을 복합 대입 연산자라고 합니다. 예를 들어, "+="는 현재 변수 값에 다른 값을 더하여 그 결과를 변수에 할당합니다.
````

## 💡비트 연산자: 이진수로 데이터를 다루는 프로그래밍의 핵심

  ````
비트 연산자는 컴퓨터 프로그래밍에서 이진수로 표현된 데이터를 다루는 데에 사용되는 중요한 도구입니다. 비트 연산자는 데이터의 비트 수준에서 조작을 수행하며, 주로 저수준 프로그래밍이나 데이터 압축, 암호화, 네트워크 프로토콜 등 다양한 분야에서 활용됩니다. 다음은 주요한 비트 연산자들에 대한 설명입니다.

비트 AND (&):
비트 AND 연산자는 두 개의 비트를 비교하여 둘 다 1일 때만 결과가 1이 되는 연산입니다. 이 연산은 비트 단위로 논리 AND 연산을 수행하며, 두 비트가 모두 1일 때만 결과가 1이 됩니다. 예를 들어, 1010 & 1100은 1000이 됩니다.

비트 OR (|):
비트 OR 연산자는 두 개의 비트를 비교하여 하나 이상이 1이면 결과가 1이 되는 연산입니다. 이 연산은 비트 단위로 논리 OR 연산을 수행하며, 두 비트 중 하나 이상이 1이면 결과가 1이 됩니다. 예를 들어, 1010 | 1100은 1110이 됩니다.

비트 XOR (^):
비트 XOR 연산자는 두 개의 비트가 서로 다를 때만 결과가 1이 되는 연산입니다. 이 연산은 비트 단위로 논리 XOR 연산을 수행하며, 두 비트가 서로 다를 때만 결과가 1이 됩니다. 예를 들어, 1010 ^ 1100은 0110이 됩니다.

비트 NOT (~):
비트 NOT 연산자는 비트를 반전시킵니다. 즉, 0은 1로, 1은 0으로 바뀝니다. 이 연산은 비트 단위로 논리 NOT 연산을 수행하며, 비트를 반전시킵니다. 예를 들어, ~1010은 0101이 됩니다.

비트 이동 연산자 (<<, >>):
비트 이동 연산자는 비트를 왼쪽이나 오른쪽으로 이동시키는 데 사용됩니다. 왼쪽 시프트 연산자(<<)는 모든 비트를 왼쪽으로 이동시키고, 오른쪽에 새로운 비트를 추가합니다. 오른쪽 시프트 연산자(>>)는 모든 비트를 오른쪽으로 이동시키고, 왼쪽에 새로운 비트를 추가합니다.
````
